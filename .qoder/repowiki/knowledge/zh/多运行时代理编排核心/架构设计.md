- **适配器模式与注册中心**：定义统一的 `AgentRuntime` 接口（`stream`, `interrupt`, `isAvailable`），并通过 `runtime/registry.ts` 维护全局运行时实例映射，支持基于配置或环境感知的动态解析（`resolveRuntime`）。
- **标准化事件契约**：建立跨运行时的通用事件模型（`RuntimeRunEvent` 与 `RuntimePermissionEvent`），将不同底层协议（如 SDK 的 tool_use、Codex 的 notifications）归一化为前端可消费的 SSE 格式，确保 UI 层与具体运行时解耦。
- **会话状态隔离与持久化**：通过 `session-store.ts` 实现按运行时 ID 隔离的会话引用管理，确保在切换代理引擎时互不干扰，并支持特定运行时（如 Codex Thread、Claude SDK Session）的状态恢复。
- **能力矩阵对齐**：引入 `capability-contract.ts` 作为单一事实来源，声明各运行时对工具（MCP/AI SDK/Bridge）的支持状态与系统提示词片段，防止不同运行时间的行为漂移。
