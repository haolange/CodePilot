# New-chat Greeting — Why

> 技术实现见 [docs/handover/new-chat-greeting.md](../handover/new-chat-greeting.md)

## 解决什么问题

旧欢迎语虽然在 6 句里随机轮换，但都是通用问句、和当下无关，读起来"死板"。希望首屏
第一句话像"活的"——知道现在几点、知道你在做项目还是跟助理聊。

## 为什么这么设计

- **拆成「问候 + 问句」而不是写死整句**：时段 × 场景若每种组合写整句会爆炸
  （4 × 3 × N）。拆开后只维护 4 句问候 + 每个场景一个小问句池，组合出多样性，也好翻译、好调。
- **场景优先级 assistant > project > general**：助理是"陪伴"语气；项目是"推进工作"语气
  并带上项目名（个人化）；都没有就回通用。优先助理，因为它最具体、最该被专门招呼。
- **首屏先显示中性句再替换**：时间和随机只有浏览器里才知道，服务器猜不准；先出一句两边
  一致的安全句，再在浏览器里换成组合句——用一次几乎看不见的切换换来"不闪、控制台干净"。
  这是对历史水合 bug 的刻意规避，不是偷懒。

## 局限与未来

- 新空白 `/chat` 页暂不识别"助理工作区"（要多查一次后端，先没接）；助理语气目前主要在
  已进入助理会话的空状态出现。需要时再补。
- 文案是品牌声音，沉淀在 i18n，随时可调；当前是**中性偏暖**的默认版。后续可按品牌语气
  统一收口（更克制 / 更俏皮）。
